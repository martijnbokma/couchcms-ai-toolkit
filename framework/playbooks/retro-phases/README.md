# Retrospective Phases

Modular workflow phases for session retrospectives and doctrine evolution.

---

## Phase Overview

| Phase | File | Purpose | Duration |
|-------|------|---------|----------|
| **0** | `context.md` | Session metadata and objectives | 2-3 min |
| **1** | `success.md` | Success pattern analysis | 5-10 min |
| **2** | `failure.md` | Failure analysis and root causes | 5-10 min |
| **3** | `impact.md` | Quantitative impact assessment | 3-5 min |
| **4** | `integration.md` | Plan doctrine updates | 5-10 min |
| **5** | `propagation.md` | Execute updates and sync | 5-10 min |
| **6** | `verdict.md` | Final assessment | 2-3 min |

**Total:** ~25-50 minutes for typical retrospective

---

## Quick Start

1. **Start:** Read `../retro.md` for overview
2. **Phase 0-6:** Follow each phase sequentially
3. **Done:** Review final verdict and readiness

---

## Phase Details

### Phase 0: Context
**Input:** Completed session
**Output:** Session metadata + objectives
**Key Actions:**
- Document session type
- List objectives
- Assess outcomes

### Phase 1: Success
**Input:** Session experience
**Output:** Success patterns + lessons
**Key Actions:**
- Identify what worked
- Extract durable lessons
- Document frameworks

### Phase 2: Failure
**Input:** Session challenges
**Output:** Failure patterns + root causes
**Key Actions:**
- Analyze failures
- Identify root causes
- Document prevention

### Phase 3: Impact
**Input:** Session results
**Output:** Metrics + quality assessment
**Key Actions:**
- Collect metrics
- Assess quality
- Document insights

### Phase 4: Integration
**Input:** Lessons learned
**Output:** Update plan
**Key Actions:**
- Plan standards updates
- Define pre-flight checks
- Plan smart operations

### Phase 5: Propagation
**Input:** Update plan
**Output:** Updated configurations
**Key Actions:**
- Execute updates
- Run sync
- Verify propagation

### Phase 6: Verdict
**Input:** Evolution status
**Output:** Final assessment
**Key Actions:**
- Assess readiness
- Document maturation
- Provide verdict

---

## Design Principles

**Simple:** Each phase is focused and concise
**Modular:** Phases are independent and reusable
**AI-Optimized:** Clear structure for AI agents
**Maintainable:** Easy to update individual phases
**Efficient:** Minimal overhead, maximum learning

## Usage Tips

**For Quick Retros:**
- Focus on Phases 1-2 (success/failure)
- Skip detailed metrics
- Quick integration plan

**For Deep Retros:**
- Complete all phases thoroughly
- Add extra examples
- Document all learnings

**For Critical Sessions:**
- Never skip Phase 4-5 (integration/propagation)
- Document all decisions
- Update all relevant files

---

## Retrospective Types

**Feature Implementation:**
- Focus on agent collaboration
- Document integration patterns
- Update agent documentation

**Bug Fix:**
- Focus on root cause analysis
- Add pre-flight checks
- Update debugging patterns

**Refactoring:**
- Focus on pattern evolution
- Document architectural decisions
- Update standards

**Code Review:**
- Focus on quality patterns
- Document review criteria
- Update review checklists
