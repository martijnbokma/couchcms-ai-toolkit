Configuration File

config/standards.md

This is your single source of truth. Edit this file to customize modules, agents, and project rules.

Tips & Best Practices

  • ✓ Always edit config/standards.md - Never edit generated files like CLAUDE.md or .cursorrules directly. They get regenerated on sync.
  • ✓ Run sync after changes - After editing standards.md, run bun run toolkit sync to update all configs.
  • ✓ Use watch mode during development - Run bun run toolkit sync --watch for auto-sync.
  • ✓ Keep standards.md focused - Only move to config/context/ when your standards.md exceeds 1000 lines.
  • ✓ Commit generated files - Commit .cursorrules, CLAUDE.md, and AGENTS.md so your team uses the same configs.

Common Pitfalls to Avoid

  • ✗ Editing generated files - Your changes will be lost on next sync. Only edit config/standards.md.
  • ✗ Forgetting to sync - After changing standards.md, run sync or your AI assistants won't see the changes.
  • ✗ Skipping bun install - If scripts fail, run cd ai-toolkit-shared && bun install first.
  • ✗ Wrong directory - Make sure you're in your project root when running toolkit commands.
  • ✗ Invalid YAML syntax - Check indentation and quotes in standards.md if sync fails.
